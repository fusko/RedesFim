Apache e NGINX

UUm Servidor Web é responsável por armazenar e transmitir informações para clientes. Essa comunicação se dá através de um usuário, que utilizando um browser, solicita informações de um servidor, o qual irá localizar as informações solicitadas na requisição do cliente, a formata e devolve ao Browser, responsável por interpretar e construir uma página, legível ao usuário.
Esta parece ser uma tarefa simples, porém, normalmente, uma pagina pode chegar a receber milhares de acessos simultâneos, para isso, o software de gerenciamento do serviço web deve ser capaz de lidar com essa grande quantia de requisições. Atualmente, existem dois principais serviços como esse: o Apache e o NGINX.

Apache

Essa ferramenta, desenvolvida em 1995 por Rob McCool é oferecida em várias plataformas, como Windows, Linux e Mac. Sendo gratuito e de código aberto, ele possui inúmeros usuários ao redor do mundo.
Devido ao seu tempo de mercado e por se tratar de um código de livre acesso, ele recebe inúmeras atualizações da comunidade, mantendo assim o seu padrão de segurança elevado, graças a isso, o software é muito bem documentado.
Seu funcionamento em módulos, permite um melhor processamento das informações e melhor gerenciamento das conexões, e apesar de ter um consumo mais alto de recursos de processamento se comparado ao NGINX, ainda oferece um bom desempenho. Somente deixando a desejar em casos de muitas requisições simultâneas.

O diretório onde ficaram servidos os arquivos pelo apache é var/www

NGINX

Assim como o Apache, o NGINX é também um software de código aberto, gratuito e disponibilizado para múltiplas plataformas. Desenvolvido em 2005 por Igor Sysoev, tinha como principal objetivo ser um servidor estável, que consumisse o mínimo de recursos de hardware possível.
Sua primorosa capacidade de poupar recursos de processamento, lhe dá uma ligeira vantagem sobre o Apache neste quesito. Para alcançar este ponto, ele não precisa abrir mão da segurança, mantendo-se no mesmo patamar de seu concorrente. Sua superioridade no quesito desempenho é facilmente percebida quando é necessária uma maior escalabilidade, principalmente quando se trata de requisições simultâneas de um site estático.

O diretório onde ficaram servidos os arquivos pelo NGIX é usr/share/nginx/www

Qual o melhor?

Essa é uma pergunta difícil de responder, pois, isso depende do cenário em que será utilizado. Se há uma menor necessidade de gerenciamento e maior busca por desempenho e escalabilidade, principalmente quando há muitas requisições paralelas, o Nginx pode lhe servir melhor. Já, caso você busque uma melhor documentação, mais versatilidade no gerenciamento e personalização, o apache pode te proporcionar mais opções e ser o que você procura. De qualquer modo, ambas as ferramentas são boas e satisfazem boa parte das aplicações no mercado.

Pocesso de instalação do Nginx

Para efetuar a instalação do NGINX em um sistema operacional linux (ubuntu), devemos seguir os passos abaixo.

instalar o nginx:

sudo apt-get update sudo apt-get upgrade sudo apt-get install nginx

liberar acesso no firewall:

sudo ufw app list sudo ufw allow 'Nginx HTTP'

verificar se o serviço do NGINX está rodando:

systemctl status nginx

para instalar através do puppet, podemos utilizar a seguinte classe:

class nginx {
package {'nginx':
ensure => "installed",
require => Class["system-update"],
}

service {'nginx':
ensure => 'running',
enable => 'true',
require => Package['nginx']
}
}

onde primeiramente garantimos a instalação do pacote nginx mantendo como dependência para sua instalação a execução da classe system-update. após garantimos que o service esteja executando desde que o pacote nginx esteja instalado.

Vale lembrar que em ambos casos o arquivo index.html poderá ser encontrado no caminho '/usr/share/nginx/www/index.html'.
